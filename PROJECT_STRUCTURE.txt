KEY FEATURES
============

Database:
  ✓ plants table
    - id, name, species, location, water_frequency_days
    - sunlight_needs (enum: low/medium/high/direct)
    - last_watered_at, last_fertilized_at (timestamps)
    - notes, image_path, created_at, updated_at

  ✓ watering_logs table
    - id, plant_id (FK), watered_by, notes
    - created_at, updated_at

Models:
  ✓ Plant
    - hasMany(WateringLog)
    - isDueForWater() - checks if last_watered_at + water_frequency_days <= today
    - isDueForFertilizer() - checks if due for fertilizer every 30 days
    - getWaterStatusAttribute() - returns status string
    - getWaterStatusCssClass() - returns CSS class for status (ok/warning/critical)

  ✓ WateringLog
    - belongsTo(Plant)

Controllers:
  ✓ PlantController - Full CRUD (index, create, store, show, edit, update, destroy)
  ✓ WateringController - store() to log watering and update plant's last_watered_at

Routes:
  ✓ GET    /plants            - index
  ✓ GET    /plants/create     - create
  ✓ POST   /plants            - store
  ✓ GET    /plants/{id}       - show
  ✓ GET    /plants/{id}/edit  - edit
  ✓ PUT    /plants/{id}       - update
  ✓ DELETE /plants/{id}       - destroy
  ✓ POST   /plants/{id}/water - log watering

Console Commands:
  ✓ php artisan plants:send-reminders
    - Finds all plants due for water today
    - Sends Slack notification via webhook
    - Scheduled for daily 8 AM (configurable in console.php)

Views:
  ✓ layouts/app.blade.php
    - Base layout with nav
    - Alert flashing (success/error)
    - Clean CSS styling (no framework)

  ✓ plants/index.blade.php
    - Grid of plant cards
    - Status indicators (green/yellow/red)
    - Quick water button
    - Edit/Delete/View actions

  ✓ plants/create.blade.php
    - Form to add new plant
    - Validation messages
    - All fields: name, species, location, water_frequency_days, sunlight_needs, image_path, notes

  ✓ plants/show.blade.php
    - Plant details with all info
    - Watering history log
    - Water form to log watering
    - Edit/Delete actions

  ✓ plants/edit.blade.php
    - Form to update plant details
    - Pre-filled with current values

Slack Notifications:
  ✓ PlantCareNotification.php - Formats Slack message
    - "🌿 Good morning! Time to water these plants:"
    - Lists plant names with locations
    - Uses Slack incoming webhooks via Guzzle

Configuration:
  ✓ .env.example - Template with all vars
  ✓ config/database.php - MySQL connection
  ✓ config/services.php - Slack webhook URL
  ✓ routes/console.php - Daily schedule at 8 AM


QUICK START
===========

1. cd /sessions/zealous-lucid-dirac/mnt/personal/casa

2. composer install
   - Installs Laravel 11, Guzzle, and all dependencies

3. cp .env.example .env && php artisan key:generate
   - Set up environment variables
   - Generate APP_KEY

4. Edit .env:
   - DB_HOST, DB_PORT, DB_DATABASE, DB_USERNAME, DB_PASSWORD
   - SLACK_WEBHOOK_URL (optional, for notifications)

5. mysql -u root -p -e "CREATE DATABASE casa;"

6. php artisan migrate
   - Creates plants and watering_logs tables

7. php artisan db:seed
   - Seeds 3 sample plants (optional)

8. php artisan serve
   - Visit http://localhost:8000


TESTING THE APP
===============

1. Add a plant:
   - Click "Add Plant"
   - Fill form with plant details
   - Click "Add Plant"

2. Mark as watered:
   - Click plant to view details
   - Fill watering form at bottom
   - Click "Mark as Watered"

3. View watering history:
   - Each plant page shows all watering logs
   - Logs show who watered and when

4. Edit plant:
   - Click "Edit" button
   - Update plant details
   - Click "Save Changes"

5. Delete plant:
   - Click "Delete" button
   - Confirm deletion


SLACK INTEGRATION
=================

1. Create webhook in Slack:
   - Slack workspace settings → Apps → Incoming Webhooks
   - Create webhook, copy URL

2. Add to .env:
   SLACK_WEBHOOK_URL=https://hooks.slack.com/services/...

3. Set up scheduler (cron job):
   * * * * * cd /path/to/casa && php artisan schedule:run >> /dev/null 2>&1

4. Test:
   php artisan plants:send-reminders
   - Should send Slack message with plants needing water today
